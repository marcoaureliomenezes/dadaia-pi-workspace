## Propósito

`dadaia-pi-workspace` exists so Pi can operate as a disciplined SDD workspace
instead of a generic coding chat. The product gives Pi sessions durable project
truth, repeatable release workflows, and safe parallelism across repositories.

## Fora de escopo (deferido a backlog)

- No Claude Code support.
- No Codex support.
- No OpenCode support.
- No generic multi-harness projection chain.
- No built-in claim of sandboxing.

## Diferencial

The product is opinionated about product truth and lifecycle, but native to Pi's
minimal core. Workflow behavior is added through Pi resources instead of by
pretending Pi has another harness's primitives.
