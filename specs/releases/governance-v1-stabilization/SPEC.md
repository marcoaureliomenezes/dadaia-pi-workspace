# SPEC - governance-v1-stabilization

**Status:** Aprovado

## Scope

Stabilize governance v1 instead of adding another open-ended governance feature wave.

Acceptance:

1. Operator documentation exists for lifecycle usage, happy path, blocked recovery, and phase commands.
2. Governance v1 is frozen with included scope, explicit out-of-scope items, and a no-new-governance-feature-without-backlog rule.
3. A regression smoke script exercises bind, spec-review, implementation-task, RC creation, QA/security/code review, readiness, evidence bundle, closure, and archive.
4. Existing generated evidence is bundled for key historical releases; current evidence remains inspectable.
5. Product memory marks governance v1 as the stable baseline.
